# 提示词 4：公共功能和权限专项验收

```text
请执行司法鉴定管理系统“公共功能和权限专项验收”。

目标：
在业务流程逐项验收的基础上，单独验证系统公共能力和权限边界。不要把公共功能混在流程里简单带过。公共功能不通过时，即使业务流程看起来能提交，也不能结项。

验收依据：
1. docs/司法鉴定系统使用手册.docx
2. docs/manual-scope-acceptance-template.md
3. docs/judicial-appraisal-master-checklist.md
4. docs/judicial-appraisal-flow-verification-matrix.md
5. docs/judicial-appraisal-manual-based-requirements-v2.md
6. docs/original-oa-samples/
7. 最新 browser validation record
8. checklist.md
9. GEMINI.md
10. AGENT.md

测试数据要求：
项目中之前已经导出过 Excel 表格数据，可以用于测试。请优先使用 docs/original-oa-samples/exports/ 下的 Excel 导出文件，结合 docs/original-oa-samples/csv-preview/、docs/original-oa-samples/json/、docs/original-oa-samples/metadata/ 和 docs/original-oa-samples/raw/，验证待办、已办、我的发起、工作查询、流程详情、附件、归档、报表和权限数据范围是否与原 OA 数据口径一致。不要优先凭空造数据；只有导出数据不足以覆盖权限或边界场景时，才补充最小必要测试数据，并说明原因。

专项验收范围：

1. 登录和会话
   - 正确账号可登录。
   - 错误密码不可登录。
   - 禁用或无效账号不可登录。
   - 登录后能获取正确用户信息。
   - 退出登录后不能继续访问受保护页面。
   - 会话过期或 token 失效处理正确。

2. 菜单权限
   - 不同角色看到的菜单正确。
   - 非授权角色看不到不该看的流程入口。
   - 直接输入 URL 不能绕过菜单权限。
   - 刷新页面后菜单仍正确。
   - 菜单与后端权限配置一致。

3. 待办、已办、我的发起
   - 待办只显示当前账号应处理的任务。
   - 已办显示当前账号已处理任务。
   - 我的发起显示当前账号发起的流程。
   - 点击列表能进入正确详情或办理页。
   - 列表状态、标题、发起人、当前节点、时间等字段正确。
   - 分页、搜索、排序可用。

4. 工作查询
   - 按案件编号、流程名称、状态、时间、发起人/办理人、部门或权限范围查询。
   - 查询结果与权限范围一致。
   - 查询字段、状态和数据口径应对照导出 Excel、CSV/JSON 预览和 metadata。
   - 详情页完整展示表单、附件、意见、日志、状态。

5. 流程详情
   - 未办结详情、办结详情、被退回详情、被终止详情、被暂停详情。
   - 详情页字段完整。
   - 详情页状态准确。
   - 详情页操作按钮符合当前节点和当前角色。
   - 办结详情只读。

6. 附件能力
   - 上传、下载、预览。
   - 删除或替换，如系统支持。
   - 多附件、大小限制、类型限制、重名文件。
   - 权限控制、后续节点继承、办结详情可查、工作查询详情可查。

7. 办理意见
   - 每个节点可填写意见。
   - 意见提交后保存。
   - 后续节点可查看历史意见。
   - 退回意见可追溯。
   - 办结详情可查看全部意见。
   - 空意见或必填意见规则符合要求。

8. 流程日志
   - 发起、提交、审核、退回、重新提交、终止、暂停、恢复、办结日志。
   - 日志中的处理人、节点、动作、时间、意见准确。

9. 退回和重新提交
   - 退回到上一节点。
   - 退回到发起人，如系统支持。
   - 退回后数据保留。
   - 退回后待办归属正确。
   - 重新提交后继续流转。
   - 退回日志和意见准确。

10. 办结只读
   - 办结后不可修改关键业务字段。
   - 办结后不可重复提交。
   - 办结后可以查看表单、附件、意见、日志。
   - 办结后工作查询详情状态正确。

11. 归档和知识库
   - 可归档流程能归档。
   - 归档后状态正确。
   - 归档材料完整。
   - 知识库能查看应归档材料。
   - 非授权角色不能越权查看归档材料。
   - 原 OA 样本中的归档习惯和字段可对照。

12. 报表或统计
   - 手册范围内的查询统计或报表能访问。
   - 数据范围正确。
   - 统计口径正确。
   - 权限范围正确。
   - 导出功能如存在则验证可用。

13. 数据权限
   - 不同角色只能看到应看到的数据。
   - 不同部门或岗位之间不能越权。
   - 经办人、负责人、领导、档案、财务等角色权限边界正确。
   - 前端隐藏和后端拦截都要验证。
   - 直接调用接口不能绕过权限。

14. 设计器或配置运行时
   - 如果流程设计器、表单设计器属于当前运行依赖，需要验证：
     表单配置保存、流程配置保存、版本启用、运行时加载、权限配置、修改配置后不破坏已有流程实例。
   - 如果不属于手册结项范围，请说明理由。

15. 不通过项处理规则
   - 公共功能属于所有流程基础能力，不通过不能结项。
   - 手册范围内问题必须当场修复并复验。
   - 权限越权、附件丢失、日志缺失、查询不准、办结可编辑等问题必须优先修复。

16. 更新验收文档
   - 更新 docs/manual-scope-acceptance-template.md。
   - 更新 docs/judicial-appraisal-master-checklist.md。
   - 更新 docs/judicial-appraisal-flow-verification-matrix.md。
   - 更新最新 browser validation record。
   - 更新 checklist.md。
   - 更新 GEMINI.md。

输出结果：
请在本次操作结束时输出以下内容：

1. 本次专项验收的公共功能清单。
2. 本次使用的导出数据：
   - Excel 文件路径。
   - CSV/JSON/metadata/raw 对照文件路径。
   - 用于哪些公共功能或权限场景。
3. 公共功能验收结果表：
   - 功能名称。
   - 使用账号。
   - 验证方式。
   - 是否通过。
   - 证据说明。
   - 当前状态：[x] / [~] / [ ]。
4. 权限验收矩阵：
   - 角色。
   - 可见菜单。
   - 可发起流程。
   - 可办理流程。
   - 可查询数据范围。
   - 可访问附件范围。
   - 是否发现越权。
5. 附件专项结果。
6. 查询和详情专项结果。
7. 日志和意见专项结果。
8. 归档、知识库、报表专项结果。
9. 本次发现的问题清单：
   - 问题编号。
   - 功能模块。
   - 问题描述。
   - 是否手册范围。
   - 是否已修复。
   - 复验结果。
10. 本次修改的文件清单。
11. 本次更新的验收文档清单。
12. 当前公共功能总体状态：
    - [x] 数量。
    - [~] 数量。
    - [ ] 数量。
13. 是否可以进入“最终结项审计”：
    - 可以 / 不可以。
    - 原因。
```
