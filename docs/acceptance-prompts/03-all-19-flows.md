# 提示词 3：19 个流程逐项验收

```text
请执行司法鉴定管理系统“19 个手册流程逐项验收”。

目标：
在主干流程已经具备运行基础后，按照手册范围逐个跑完全部业务流程。每个流程都必须使用真实业务账号、真实页面、真实流程实例完成闭环验收。发现手册范围内的问题必须当场修复并复验。

验收依据：
1. docs/司法鉴定系统使用手册.docx
2. docs/manual-scope-acceptance-template.md
3. docs/judicial-appraisal-master-checklist.md
4. docs/judicial-appraisal-flow-verification-matrix.md
5. docs/judicial-appraisal-manual-based-requirements-v2.md
6. docs/judicial-appraisal-flowchart-baseline.md
7. docs/original-oa-samples/
8. 最新 browser validation record
9. checklist.md
10. GEMINI.md
11. AGENT.md

测试数据要求：
项目中之前已经导出过 Excel 表格数据，可以用于测试。请优先使用 docs/original-oa-samples/exports/ 下的 Excel 导出文件作为 19 个流程验收的测试数据来源；每个流程都应优先匹配同名或相关导出文件，并结合 docs/original-oa-samples/csv-preview/、docs/original-oa-samples/json/、docs/original-oa-samples/metadata/ 和 docs/original-oa-samples/raw/ 做字段、流程名称、状态、查询结果和原 OA 等价性对照。不要优先凭空造数据；只有导出数据无法覆盖某个分支或边界场景时，才补充最小必要测试数据，并说明原因。

验收范围：
请逐项验收以下 19 个流程：

1. 收到委托书
2. 不予受理
3. 初步勘验
4. 发交费通知书及相关函件
5. 材料接收与返还
6. 鉴定意见书征求意见稿送审稿编制
7. 鉴定意见书送审稿编制
8. 出具鉴定意见书
9. 编制内部质量控制文件
10. 现场勘验
11. 终止鉴定
12. 收到撤案函
13. 案件暂停
14. 退费
15. 归档
16. 收到法院其他函件，含异议函
17. 收到出庭通知
18. 用章申请表
19. 财务报销

每个流程必须验证以下内容：

1. 发起入口
   - 菜单入口是否正确。
   - 是否能从真实页面发起。
   - 如果流程应由前置流程触发，验证触发条件是否正确。
   - 非授权角色是否无法发起。

2. 表单字段
   - 字段是否覆盖手册要求。
   - 字段是否能对应导出 Excel、CSV/JSON 预览和 metadata 中的历史字段。
   - 必填规则是否正确。
   - 下拉、日期、人员、部门、金额、文本等控件是否可用。
   - 保存后数据是否回显。
   - 提交后数据是否传递到后续节点。

3. 节点和角色
   - 流程节点顺序是否符合手册。
   - 每个节点处理角色是否正确。
   - 待办是否出现在正确账号下。
   - 错误角色是否无法办理。
   - 下一步处理人是否正确。

4. 分支条件
   - 受理/不受理。
   - 需要/不需要勘验。
   - 通过/退回。
   - 继续/终止。
   - 暂停/恢复。
   - 退费/不退费。
   - 用章/不用章。
   - 归档/不归档。
   - 其他手册中定义的关键分支。
   - 每个流程至少覆盖主要正向路径和必要反向路径。

5. 附件
   - 上传、下载、预览。
   - 删除或替换，如系统支持。
   - 后续节点可见。
   - 办结详情可见。
   - 工作查询详情可见。
   - 权限外账号不可越权访问。

6. 办理意见和流程日志
   - 每个节点办理意见是否可填。
   - 办理意见是否保存。
   - 流程日志是否记录节点、处理人、处理时间、动作、意见。
   - 退回和重新提交是否记录准确。

7. 退回和重提
   - 至少验证关键流程的退回路径。
   - 退回后原办理人是否可重新办理。
   - 退回后数据是否保留。
   - 重新提交后流程是否继续正确流转。

8. 办结和只读
   - 流程能否正常办结。
   - 办结状态是否正确。
   - 办结详情是否只读。
   - 办结后是否能从工作查询进入详情。
   - 办结后附件、意见、日志是否完整。

9. 工作查询
   - 按流程名称、案件编号、状态、发起人或办理人、时间查询。
   - 查询结果字段是否与手册、导出 Excel 和原 OA 样本基本一致。
   - 详情页是否完整。

10. 原 OA 样本对照
   - 对照 docs/original-oa-samples/exports/、csv-preview、json、metadata、raw。
   - 判断是否存在关键业务偏差。
   - 如果存在偏差，判断是否需要修复。

11. 不通过项处理规则
   - 属于手册范围的问题必须当场修复。
   - 修复后必须复验。
   - 不允许将流程闭环失败、权限错误、附件错误、日志缺失、查询缺失、归档缺失归入“后续优化”。

12. 更新验收文档
   - 每完成一个流程，都要同步更新：
     docs/manual-scope-acceptance-template.md
     docs/judicial-appraisal-master-checklist.md
     docs/judicial-appraisal-flow-verification-matrix.md
     最新 browser validation record
     checklist.md
     GEMINI.md

执行方式：
请不要只做静态代码检查。必须尽可能启动系统并通过浏览器或接口完成真实运行验收。
如果某个流程暂时无法完整验收，请说明阻塞原因，并先修复导致无法验收的手册范围问题。

输出结果：
请在本次操作结束时输出以下内容：

1. 本次实际验收了哪些流程。
2. 每个流程使用的导出数据：
   - Excel 文件路径。
   - CSV/JSON/metadata 对照文件路径。
   - 选取的案件或样本记录。
   - 用于字段、状态、查询或流程等价性对照的内容。
3. 每个流程的验收结果表：
   - 流程名称。
   - 发起方式。
   - 使用账号。
   - 流程实例编号或案件编号。
   - 是否跑到办结。
   - 是否验证附件。
   - 是否验证退回。
   - 是否验证日志。
   - 是否验证工作查询。
   - 是否验证权限。
   - 是否对照原 OA 样本。
   - 最终状态：[x] / [~] / [ ]。
4. 每个流程的主要证据：
   - 关键节点。
   - 处理账号。
   - 办理动作。
   - 查询结果。
   - 详情页验证结果。
5. 本次发现的问题清单：
   - 问题编号。
   - 所属流程。
   - 问题描述。
   - 影响范围。
   - 是否手册范围。
   - 是否已修复。
   - 复验结果。
6. 本次完成的修复清单：
   - 文件。
   - 修改内容。
   - 对应问题。
   - 复验结果。
7. 本次未完成或仍为 [~] / [ ] 的流程：
   - 流程名称。
   - 卡点。
   - 下一步需要做什么。
8. 本次更新的验收文档清单。
9. 当前 19 个流程总体进度：
   - [x] 数量。
   - [~] 数量。
   - [ ] 数量。
10. 是否可以进入“公共功能和权限专项验收”：
   - 可以 / 不可以。
   - 原因。
```
