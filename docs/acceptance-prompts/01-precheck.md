# 提示词 1：验收前置清点

```text
请对当前司法鉴定管理系统做一次“手册级验收前置清点”。

目标：
在正式跑全流程前，确认系统是否已经具备手册级验收条件，避免后续流程验收卡在基础问题上。

验收依据：
1. docs/司法鉴定系统使用手册.docx
2. docs/manual-scope-acceptance-template.md
3. docs/judicial-appraisal-master-checklist.md
4. docs/judicial-appraisal-flow-verification-matrix.md
5. docs/judicial-appraisal-manual-based-requirements-v2.md
6. docs/original-oa-samples/
7. checklist.md
8. GEMINI.md
9. AGENT.md

测试数据要求：
项目中之前已经导出过 Excel 表格数据，可以用于测试。请优先使用 docs/original-oa-samples/exports/ 下的 Excel 导出文件作为验收测试数据来源，并结合 docs/original-oa-samples/csv-preview/、docs/original-oa-samples/json/、docs/original-oa-samples/metadata/ 和 docs/original-oa-samples/raw/ 做字段、流程名称、状态、查询口径和原 OA 等价性对照。不要优先凭空造数据；只有导出数据不足以覆盖某个场景时，才补充最小必要测试数据，并说明原因。

请完成以下工作：

1. 阅读并梳理当前验收依据
   - 明确当前系统的手册范围。
   - 明确 19 个业务流程。
   - 明确公共功能范围。
   - 明确当前 checklist、master checklist、flow matrix 中的状态口径。
   - 明确哪些文档是当前有效入口，哪些只是 archive 历史参考。

2. 检查系统启动条件
   - 检查后端启动方式、配置文件、端口、数据库连接。
   - 检查前端启动方式、环境变量、API 地址。
   - 尝试启动后端和前端。
   - 如果已经有服务在运行，确认服务是否可访问。
   - 记录启动命令、访问地址和异常信息。

3. 检查账号和角色
   - 确认非 admin 业务账号是否存在。
   - 至少检查以下角色或账号是否可用于验收：
     case_acceptor、project_leader、project_assistant、dept_leader、tech_leader、director_review、archivist、center_archivist、business_staff、finance。
   - 验证这些账号是否能登录。
   - 验证登录后菜单是否与角色匹配。
   - 如果账号、密码、权限、菜单存在问题，请当场修复手册范围内的问题，并复验。

4. 检查流程和表单基础配置
   - 确认 19 个手册流程是否存在流程定义。
   - 确认每个流程是否有关联表单或办理页面。
   - 确认流程是否可发起，或可由前置流程触发。
   - 确认流程节点、处理角色、分支条件是否至少具备运行基础。
   - 如果流程定义缺失、流程无法发起、表单缺失、节点处理人错误，请记录并优先修复。

5. 检查公共功能基础能力
   - 待办、已办、我的发起、工作查询、流程详情。
   - 附件上传、下载、预览。
   - 办理意见、流程日志、退回、办结只读。
   - 归档、知识库、报表或统计。
   - 数据权限、菜单权限。

6. 建立或确认验收种子数据
   - 优先从 docs/original-oa-samples/exports/ 选取导出的 Excel 案件数据。
   - 对照 CSV/JSON/metadata/raw 样本，确认流程名称、字段、状态、查询习惯是否有可比对数据。
   - 如果缺少种子数据，请补充最小可用测试数据，至少满足后续主干流程试跑。

7. 不通过项处理规则
   - 如果发现属于手册范围内的问题，不要只记录为“后续优化”。
   - 能当场修复的必须当场修复。
   - 修复后必须复验。
   - 只有非手册范围、非结项必要项，才允许归入后续优化。

输出结果：
请在本次操作结束时输出以下内容：

1. 本次读取了哪些验收依据文件。
2. 本次使用或确认可使用的导出数据：
   - Excel 文件路径。
   - 对应流程。
   - 可用于哪些测试场景。
   - 是否需要补充测试数据。
3. 后端启动状态：
   - 启动命令。
   - 访问地址。
   - 是否成功。
   - 如果失败，失败原因和处理结果。
4. 前端启动状态：
   - 启动命令。
   - 访问地址。
   - 是否成功。
   - 如果失败，失败原因和处理结果。
5. 业务账号检查结果：
   - 账号。
   - 角色。
   - 是否可登录。
   - 菜单是否正确。
   - 是否存在权限问题。
6. 19 个流程基础配置检查表：
   - 流程名称。
   - 是否存在。
   - 是否可发起或可触发。
   - 是否有关联表单。
   - 是否有明显节点/角色问题。
   - 当前状态：[x] / [~] / [ ]。
7. 公共功能基础能力检查表：
   - 功能名称。
   - 是否可访问。
   - 是否可运行。
   - 是否存在问题。
8. 本次发现的问题清单：
   - 问题编号。
   - 问题描述。
   - 是否属于手册范围。
   - 是否已修复。
   - 复验结果。
9. 本次修改了哪些文件。
10. 本次新增或更新了哪些验收记录。
11. 下一步是否可以进入“主干流程试跑”：
   - 可以 / 不可以。
   - 原因。
   - 如果不可以，还需要先完成什么。
```
