### 2. 草稿创建与删除

使用 `case_acceptor`：

1. 登录后进入 `/case/new`。
2. 点击“收到委托书”创建草稿。
3. 页面跳转到 `/case/2284`。
4. 进入 `/work-query?caseStatus=DRAFT&keyword=收到委托书`。
5. 草稿行显示“删除草稿”。
6. 点击“删除草稿”，弹出确认框。
7. 点击确认框中的“删除”。
8. 页面提示“草稿已删除”。
9. 再访问 `/api/cases/2284`，后端返回 `code=404`、`message=案件不存在或已删除`。

结论：草稿删除前端入口可用，且后端删除后不可再查询。

### 3. 工作查询详情可打开

使用 `case_acceptor`：

1. 登录后进入 `/work-query`。
2. 点击列表第一条“查看详情”。
3. 页面跳转到 `/case/2283?from=/work-query&fromLabel=工作查询`。
4. 详情页展示“收到委托书”、草稿状态、案件信息、动态表单字段。
5. 页面无“系统异常”。

结论：工作查询详情入口可打开，不是空页或 404。

### 4. 待办工作可进入办理页

5. 详情页展示真实案号 `JA-2067`、当前环节“项目负责人决策”、当前任务“收到委托书 - 项目负责人决策”、案件信息和动态表单。

结论：待办工作已携带 `taskId` 进入真实办理上下文，并可通过浏览器页面提交流转。
