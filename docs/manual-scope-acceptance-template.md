# 司法鉴定系统手册范围总体验收模板

用途：本模板用于验收《司法鉴定系统使用手册》范围内的全部流程和公共功能。只有本模板覆盖的 19 个业务流程、公共功能、权限、附件、意见、日志、查询、归档、报表、知识库和全量回归均通过，且阻塞缺陷关闭，系统才可以进入结项判断。

本模板不是单个流程的验收记录，而是整套系统的手册范围总体验收底稿。单个流程的证据可附在本文件对应章节，或链接到专项浏览器/Playwright 验收记录。

执行原则：验收不是旁观式打分。凡属于手册范围内的流程或公共功能，验收时发现不通过项，应当场定位原因并修改代码、配置、数据或文档；修改后必须立即复验同一验收点，并把修复前结果、修复动作、复验结果和证据编号记录在本模板中。只有确认该项不属于手册范围、受外部条件阻塞且无法在当前环境处理，或修复会影响更高优先级验收时，才允许暂列阻塞项。

## 0. 验收口径

状态定义：

- `[x]`：已按《司法鉴定系统使用手册》完成真实页面验收，覆盖非 admin 业务账号、待办办理、办结只读、工作查询详情、附件、办理意见、流程日志、退回、必要分支和归档/查询证据。
- `[~]`：配置、入口、后端运行时测试或部分页面链路具备，但缺完整真实页面验收、非 admin 全链路、附件/意见/日志/查询闭环或手册要求的关键能力。
- `[ ]`：未实现、未测试，或没有有效运行级证据。

禁止把以下证据单独作为 `[x]` 依据：

- 流程配置存在；
- 表单配置存在；
- 菜单或入口可见；
- 子流程被触发；
- 测试类存在；
- Service 层、MockMvc 或 API 测试通过；
- 使用 `admin` 或管理员 ID 代替业务账号；
- 手工构造 `CaseTask`、`CaseWfInstance` 后完成流程；
- 只跑主路径，没有覆盖退回、附件、意见、日志、待办、办结、工作查询和权限隔离。

验收失败处理规则：

1. 发现失败项后，先判断是否属于《司法鉴定系统使用手册》范围。
2. 属于手册范围的，必须当场修复；不能只记录为后续优化。
3. 修复可以包括前端页面、后端接口、流程配置、表单配置、权限菜单、测试、验证记录和 checklist 状态。
4. 修复后必须复验原失败路径，并补充回归相关联路径。
5. 复验通过前，该流程或功能不得标记 `[x]`。
6. 若无法当场修复，必须在第 9 节记录阻塞原因、影响范围、临时状态和下一步动作。

手册级完成最小定义：

```text
使用手册步骤 + 真实业务账号 + 真实浏览器页面 + 完整流程节点
+ 附件/意见/日志/退回/查询/办结/归档 + 可复查证据
```
