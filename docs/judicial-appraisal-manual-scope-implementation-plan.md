# 司法鉴定管理系统 (使用手册范围版) 执行计划

更新时间：2026-06-12

## 1. 核心目标

完美实现《司法鉴定系统使用手册.docx》全部功能。当前项目不再以完整 OA 重构为必要目标，所有资源优先投入司法鉴定 19 个业务流程的运行级闭环。

后续运行级闭环主要按照 `docs/judicial-appraisal-flowchart-baseline.md` 登记的主流程图与细化流程图逐项跑通。流程图用于校验节点顺序、角色、判断条件、退回路径、输出文件和父子流程关系；与手册正文发生冲突时，以手册正文范围为准并记录差异。

## 2. 必须实现的页面与功能

### 2.1 门户与导航
- **主界面**：按手册 2.1 节，包含常用工作、流程中心、数据报表、知识库。
- **新建工作**：按手册 2.2 节，区分“直接发起”、“关联发起”、“系统触发”。
- **我的工作**：待办、已办、办结列表，支持状态、类型、发起人搜索。
- **工作查询**：支持多维度组合查询（部门、人员、案件编号、状态、日期范围）。

### 2.2 通用流程办理页
- **角色隔离**：实现“主办人”与“经办人”权限分离，经办人默认只读。
- **必填校验**：后端需严格按发布版本的表单 schema 校验，不满足必填项无法转交。
- **附件去重**：同一环节/同一案件禁止上传相同文件 MD5 的附件。
- **会签意见**：支持历史环节审批意见的聚合展示。
- **流程图与日志**：实时展示当前流转位置及详细操作轨迹。
- **自动归档**：节点提交时，表单快照与附件必须自动进入案件归档目录。

## 3. 19 个验收流程清单

| 序号 | 流程编码 | 流程名称 | 核心验收点 |
|---:|---|---|---|
| 1 | `received-entrust` | 收到委托书 | 收案登记、审阅决策、并行子流程分发 |
| 2 | `preliminary-survey` | 初步勘验 | 方案、出入库、鉴定条件判定 |
| 3 | `payment-notice` | 发交费通知书及相关函件 | 函件编制、审核用章、缴费确认 |
| 4 | `quality-control` | 编制内部质量控制文件 | F 类项目判断、多级审核、用章 |
| 5 | `field-survey` | 现场勘验 | 方案、勘验记录、设备归还、金额审核 |
| 6 | `material-receive-return` | 材料接收与返还 | 登记、补材、保管、返还 |
| 7 | `draft-opinion-review` | 鉴定意见书征求意见稿送审编制 | 征求意见稿初稿、三级串行审核、定稿流向 |
| 8 | `final-opinion-review` | 鉴定意见书送审稿编制 | 意见书初稿、三级串行审核、确定版本 |
| 9 | `issue-opinion` | 出具鉴定意见书 | 承诺书、并行盖章/开票汇聚、送达归档 |
| 10 | `issue-draft-opinion` | 出具征求意见稿 | 说明函、用章、送达、反馈流转 |
| 11 | `court-letter` | 收到法院其他函件（含异议函） | 登记、异议判断、回复函、用章 |
| 12 | `court-appearance` | 收到出庭通知 | 费用通知、准备、登记、归档 |
| 13 | `reject-acceptance` | 不予受理 | 通知书编制、用章、送达归档 |
| 14 | `withdraw-case-letter` | 收到撤案函 | 登记、退费/终止鉴定判断 |
| 15 | `refund` | 退费流程 | 申请、财务确认、金额回滚 |
| 16 | `expense-reimbursement` | 财务报销 | 报销登记、财务处理、支付结果 |
| 17 | `terminate-appraisal` | 终止鉴定 | 终止函编制、用章、归档 |
| 18 | `archive` | 归档 | 邮寄入库/中心审核两条路径、正式入库 |
| 19 | `seal-application` | 用章流程 | 申请、审核、盖章件回传 |

## 4. 当前完成情况 assessment

- **底座 (Phase 1-3)**: 基本完成。RBAC、数据权限、动态表单/流程引擎、文件归档、审计已可用。
- **高保真 (Phase 4)**: 首轮配置已导入，部分主链 E2E 已跑通。
- **Gap**: 仍需逐一针对 19 个流程进行手册级精度对齐，尤其是角色权限（主办/经办）和导出报表的细节。
