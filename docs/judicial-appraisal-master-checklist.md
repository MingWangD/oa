## 1. 文件定位

本文件是后续所有司法鉴定系统功能实现、测试补齐、验收矩阵更新和协作交接的总执行清单。

后续工作必须先对照本清单，再进入具体代码修改或测试编写。若本清单与更高优先级资料冲突，以更高优先级资料为准，并同步修订本清单。

## 2. 依据优先级

1. `docs/司法鉴定系统使用手册.docx`
2. `exports/司法鉴定系统使用手册.txt`
3. `docs/judicial-appraisal-flowchart-baseline.md` 与 `docs/flowcharts/` 三张流程图
4. `docs/judicial-appraisal-manual-based-requirements-v2.md`
5. 当前代码、数据库和已发布流程/表单配置
6. 原有 OA 网站中的真实案件与流程数据，仅作测试数据、样板对比和等价性校验来源
7. 旧 Markdown 文档，仅作历史参考

说明：

- `GEMINI.md` 用于 Codex 与 Gemini 对齐协作进度。
- `README.md` 用于项目组成员了解项目定位、运行方式和验收范围。
- 本文件用于落地执行，判断下一步该补什么、怎样才算完成。
- 本轮校准结论：后端运行级测试、流程配置、子流程触发和入口可见均不能直接等同于手册级完成。手册级完成必须有非 admin 业务账号真实页面验收，覆盖待办办理、办结只读、工作查询详情、附件、办理意见、流程日志、退回和必要分支。

## 4. 三张流程图使用顺序

后续每项流程工作按以下顺序校验：

1. `docs/flowcharts/judicial-appraisal-main-flow.png`
   - 用途：确定主流程骨架、入口、出口、父子流程关系、主链闭环。
   - 必查：委托、受理、初勘、缴费、质控、勘验、材料、意见书、出具、归档、财务和结束状态。

2. `docs/flowcharts/judicial-appraisal-detailed-flow-overview.png`
   - 用途：补齐节点、角色、附件、退回线、判断条件、输出文件、后续流向。
   - 必查：每个流程的主路径、分支路径、退回路径、终止路径、归档路径。

3. `docs/flowcharts/final-opinion-review-detailed-flow.png`
   - 用途：专项校验“鉴定意见书送审稿编制”。
   - 必查：项目辅助人、项目负责人、技术负责人、部门负责人串行审核；版本稿件留痕；审核不得合并。

## 5. 使用手册公共功能清单

本轮反查结论：第五部分原有若干 `[x]` 只能证明页面入口、局部控件或单路径可用，不能证明已按手册完成公共功能验收。除数据报表、知识库等仍为 `[ ]` 的未完成项外，已将有局部证据但缺完整浏览器验收的项目统一降级为 `[~]`。

### 5.1 主界面与导航

- [~] 登录后进入主界面。
- [ ] 左侧菜单栏包含快捷菜单、个人事务、流程中心、知识管理。
- [~] 流程中心承载流程创建、处理和查询。
- [ ] 知识管理通过知识库管理上传文件和流程归档情况。
- [~] 上方快捷访问页签可在已打开功能页之间切换。
- [~] 右上角用户中心支持个人信息入口和注销；页面可见，注销按钮自动化复验仍需补一次人工/浏览器确认。

### 5.2 新建工作

- [~] 能进入流程中心 -> 新建工作。

- [~] 已断言任务节点、子流程状态、父子流程关系和关键表单数据持久化。

### 6.18 归档 [~]

状态：部分完成。`ArchiveBranchVerificationTest` 覆盖后端邮寄/中心审核/正式入库路径，但测试使用管理员身份、Service 直调和手工任务构造；知识库/档案库页面、归档附件、归档记录、归档后只读、工作查询、批量下载等手册级闭环未验收。当前应标为：`[~] 部分完成，手册级归档闭环未完整验收`。

- [~] 可由其他流程流转进入。
- [~] 初始节点到达档案管理员。
- [~] 档案管理员整理项目档案、上传纸质扫描件、登记电子归档地址。
- [~] 确认是否需要邮寄，支持“直接中心审核”与“邮寄入库”分支。
- [~] 需要邮寄时转交邮寄人员填写快递单号。
- [~] 中心档案管理员审核，支持审核通过与退回复核路径。
- [~] 存放档案室或正式入库。
- [~] 结束流程，案件状态自动更新为 COMPLETED 终态。
- [~] 断言关键表单数据持久化（项目档案、扫描件、邮寄单号、存放位置等）。
- [~] 备注：本轮完成后端运行级测试，暂不包含知识库批量下载/页面美化验收。

### 6.19 用章申请表 [~]

状态：部分完成。`SealApplicationBranchVerificationTest` 覆盖后端独立发起、父流程等待与退回路径，但测试使用管理员身份、Service 直调和手工任务构造；真实页面的独立发起、父流程触发、扫描件上传/回传、办结查看和工作查询详情未完整验收。当前应标为：`[~] 子流程触发和部分后端链路已验证，独立发起和完整用章页面闭环未验收`。

- [~] 一般由其他流程在申请盖章节点中发起（子流程触发）。
- [~] 也可以单独发起，已验证 `seal-application` 独立运行全路径。
- [~] 发起者填写申请文件名、原因、用章方式并上传附件材料。
- [~] 档案管理员审核通过或退回申请人（已验证退回线）。
- [~] 盖章经办人完成盖章，登记结果。
- [~] 盖章办理完成后回传盖章扫描件，主流程唤醒继续（已验证父流程等待与唤醒）。
- [~] 已断言任务节点、流程状态、父子关系及关键表单数据持久化。
