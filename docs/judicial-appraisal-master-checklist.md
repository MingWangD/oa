# 司法鉴定系统手册与流程图总执行清单

更新时间：2026-06-13

## 1. 文件定位

本文件是后续所有司法鉴定系统功能实现、测试补齐、验收矩阵更新和协作交接的总执行清单。

后续工作必须先对照本清单，再进入具体代码修改或测试编写。若本清单与更高优先级资料冲突，以更高优先级资料为准，并同步修订本清单。

## 2. 依据优先级

1. `docs/司法鉴定系统使用手册.docx`
2. `exports/司法鉴定系统使用手册.txt`
3. `docs/judicial-appraisal-flowchart-baseline.md` 与 `docs/flowcharts/` 三张流程图
4. `docs/judicial-appraisal-manual-based-requirements-v2.md`
5. 当前代码、数据库和已发布流程/表单配置
6. 原有 OA 网站中的真实案件与流程数据，仅作测试数据、样板对比和等价性校验来源
7. 旧 Markdown 文档，仅作历史参考

说明：

- `GEMINI.md` 用于 Codex 与 Gemini 对齐协作进度。
- `README.md` 用于项目组成员了解项目定位、运行方式和验收范围。
- 本文件用于落地执行，判断下一步该补什么、怎样才算完成。

## 3. 测试数据来源

原有 OA 网站中的真实案件、流程、附件、归档和报表数据可以作为测试数据来源，用于构造更贴近实际业务的验收样例。

原有 OA 测试环境：

- 地址：`http://123.56.9.140:28080/`
- 登录账号信息由项目负责人线下提供，不写入仓库文档或测试代码明文。
- 取样时优先使用原有 OA 自带的查询、导出、报表导出、附件批量下载等能力；无法导出时，再通过页面取样形成手工记录。
- 已沉淀的工作查询样例位于 `docs/original-oa-samples/`，包含原始 `.xls`、接口 JSON、CSV 预览和导出元数据。

使用规则：

- [ ] 可用原有 OA 网站中的案件号、委托人、法院函件、鉴定事项、金额、材料、附件名称、流程节点和办理意见构造测试样例。
- [ ] 可用原有 OA 网站中的完整案件流转记录做等价性校验。
- [ ] 可用原有 OA 网站中的归档目录、文件命名、附件版本、邮寄记录、出庭记录、退费记录、终止记录做数据结果对比。
- [ ] 可用原有 OA 网站中的报表样例核对查询、统计和导出字段。
- [ ] 优先通过原有 OA 的导出功能获取测试样例，保留导出条件、导出时间、导出文件字段说明和样例用途。
- [ ] 原有 OA 网站只作为测试数据和结果对照来源，不作为需求优先级高于使用手册或流程图的依据。
- [ ] 原有 OA 网站中超出使用手册范围的模块，不得因此扩展为当前必要验收范围。
- [ ] 使用真实数据编写自动化测试时，应脱敏姓名、手机号、身份证号、地址、案号中敏感片段和附件内容。
- [ ] 若原有 OA 网站样例与使用手册或流程图不一致，先记录差异，再按使用手册优先处理。
- [ ] 若需要登录原有 OA 网站取样，取样结果应沉淀为本项目可复用的测试夹具、SQL seed、接口样例或人工验收记录。

## 4. 三张流程图使用顺序

后续每项流程工作按以下顺序校验：

1. `docs/flowcharts/judicial-appraisal-main-flow.png`
   - 用途：确定主流程骨架、入口、出口、父子流程关系、主链闭环。
   - 必查：委托、受理、初勘、缴费、质控、勘验、材料、意见书、出具、归档、财务和结束状态。

2. `docs/flowcharts/judicial-appraisal-detailed-flow-overview.png`
   - 用途：补齐节点、角色、附件、退回线、判断条件、输出文件、后续流向。
   - 必查：每个流程的主路径、分支路径、退回路径、终止路径、归档路径。

3. `docs/flowcharts/final-opinion-review-detailed-flow.png`
   - 用途：专项校验“鉴定意见书送审稿编制”。
   - 必查：项目辅助人、项目负责人、技术负责人、部门负责人串行审核；版本稿件留痕；审核不得合并。

## 5. 使用手册公共功能清单

### 5.1 主界面与导航

- [ ] 登录后进入主界面。
- [ ] 左侧菜单栏包含快捷菜单、个人事务、流程中心、知识管理。
- [ ] 流程中心承载流程创建、处理和查询。
- [ ] 知识管理通过知识库管理上传文件和流程归档情况。
- [ ] 上方快捷访问页签可在已打开功能页之间切换。
- [ ] 右上角用户中心支持个人信息入口和注销。

### 5.2 新建工作

- [ ] 能进入流程中心 -> 新建工作。
- [ ] 能选择司法鉴定栏。
- [ ] 能展示司法鉴定相关子任务。
- [ ] 正常主流程从“收到委托书”快速新建。
- [ ] 后续子流程由节点操作和条件分支自动流转，不要求每步手动新建。
- [ ] 收到异议函、收到出庭通知等主流程后续事项可手动新建。
- [ ] 手动新建后续流程时必须支持关联先前案件或流程。

### 5.3 我的工作

- [ ] 支持待办工作、办结工作切换。
- [ ] 待办工作展示流转到当前用户的待处理任务。
- [ ] 办结工作展示本人经办并办结的任务。
- [ ] 支持按工作名和案件号搜索。
- [ ] 列表展示流程相关信息。
- [ ] 支持表格宽度调整或等价的表格展示体验。
- [ ] 当前用户只能办理流转给自己的任务。

### 5.4 工作查询

- [ ] 支持按案件号、工作名等关键信息搜索。
- [ ] 支持范围选项。
- [ ] 有权限用户可查询其他人经办事项。
- [ ] 无权限用户不得越权查看不应查看的数据。

### 5.5 数据报表

- [ ] 根据真实流程表单填写情况自动收集统计信息。
- [ ] 能展示所需统计数据。
- [ ] 右上角提供导出选项。
- [ ] 支持导出方式选择。
- [ ] 支持每页展示数据条数设置。
- [ ] 报表数据随流程办理结果更新。

### 5.6 知识库

- [ ] 左侧文件夹树可用。
- [ ] 点击文件夹后展示目录下内容。
- [ ] 按案件号形成案件文件夹。
- [ ] 案件号文件夹内包含每一步流程结果。
- [ ] 支持手动上传。
- [ ] 支持批量下载。
- [ ] 支持单击边缘选中文件。
- [ ] 支持单击中心打开文件。
- [ ] 单项打开后可查看、下载归档信息。
- [ ] 不得出现错案号归档、漏归档、重复归档不可识别。

### 5.7 通用流程办理页

- [ ] 顶部展示当前流程流水号。
- [ ] 顶部展示流程名称。
- [ ] 流程名称从“案件号”“委托人”等表单内容生成或更新。
- [ ] 支持转交下一步。
- [ ] 支持退回。
- [ ] 支持保存。
- [ ] 支持返回。
- [ ] 保存不触发流转。
- [ ] 转交下一步触发表单必填校验。
- [ ] 带 `*` 字段为空时禁止转交。
- [ ] 附件中不能添加重复文件。
- [ ] 流程关联区展示关联流程。
- [ ] 会签意见区支持备注信息并在流程步骤中展示。
- [ ] 流转区可选择下一步流转节点、主办人、经办人。
- [ ] 主办人有权办理流程。
- [ ] 经办人收到流程但无权操作流程。
- [ ] 流程图入口可查看图形视图、列表视图、流程日志、流程设计图。
- [ ] 退回只能退到允许退回的前序节点，并保留原因。

- [ ] 已对照使用手册原文核对入口、节点、角色、附件、判断条件、输出物。
- [ ] 已对照主流程图核对父子流程关系、主链入口出口、最终状态。
- [ ] 已对照细化流程图核对分支、退回线、并行/汇聚、后续流向。
- [x] 若为鉴定意见书送审稿编制，已额外对照专项细化流程图。
- [ ] 已编写运行时测试，不仅检查配置存在。
- [ ] 测试覆盖主路径。
- [ ] 测试覆盖关键条件分支。
- [ ] 测试覆盖退回路径。
- [ ] 测试覆盖终止、归档或后续子流程路径。
- [ ] 测试核对任务状态。
- [ ] 测试核对案件状态。
- [ ] 测试核对子流程状态。
