# 司法鉴定业务管理系统 - 完整本地测试指南 (Final Test Guide)

欢迎参与司法鉴定业务管理系统的本地测试！本指南为您提供详细的“傻瓜式”走查步骤，包含所有 7 个核心业务场景的模拟数据与期待结果，帮助您完整验证系统的流程引擎流转。

## 1. 环境与预置账号说明

### 1.1 启动服务
*   **后端**：在 `judicial-appraisal-backend` 目录下执行 `mvn spring-boot:run` (端口 8080)
*   **前端**：在 `judicial-appraisal-frontend` 目录下执行 `npm run dev` (端口 5173)

### 1.2 测试账号一览 (密码统一为: `123456`)
| 账号 (用户名) | 角色及职能 | 账号 (用户名) | 角色及职能 |
| :--- | :--- | :--- | :--- |
| `admin` | 管理员、外部发起人 | `archivist` | 档案管理员 |
| `case_acceptor1` | 收案员、邮寄员 | `archivist` | 盖章经办人 |
| `dept_leader2` | 部门负责人、审批人 | `finance` | 财务人员 |
| `project_leader2`| 项目负责人、主鉴定人 | `center_archivist` | 中心档案管理员 |
| `project_assistant2`| 项目辅助人、初稿起草人 | `business_staff` | 综合业务人员、普通员工 |

*(测试建议：准备普通窗口与无痕/隐私窗口各一个，以方便在不同角色之间来回切换处理任务)*

## 3. 核心业务流程全景走查 (7 大场景)

> ⚠️ **前提说明**：在以下场景测试中，所有“发起案件”的入口都在 `admin` 账号（或其他带有申请人/发起者角色的账号）的左侧菜单**“我的工作 / 新建工作”**中。子流程的独立发起则在左侧业务菜单。

### 场景 1：不予受理流程
*   **测试目的**：验证案件在初期被部门负责人拒绝受理后，自动出具退案函并结案的逻辑。

| 步骤 | 节点名称 | 登录账号 | 关键填写表单数据与选择 | 期待流转结果 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | 发起案件 | `admin` | **标题**: `退案测试`<br>**案号**: `(2026)民初001号`<br>**委托人**: `张三`<br>**金额**: `5000`<br>**鉴定类别**: `法医临床`<br>**紧急程度**: `普通`<br>点击【提交流转】。 | 流转至收案员。 |
| **2** | 登记与审查 | `case_acceptor1` | 确认基本信息无误，直接点击【提交流转】。 | 流转至部门负责人。 |
| **3** | 受理审批 | `dept_leader2`| **委托审查是否受理**: 选择 **`否 (false)`**<br>**不予受理原因**: 填写 `材料严重不足，无法鉴定`。<br>点击【提交流转】。 | **触发分支**：流程直接跳过后续所有节点，进入生成退案函与结案归档步骤。 |

### 场景 2：F类质控文件审核与用章驳回退回
*   **测试目的**：验证内部质控审核及在物理盖章环节被退回重新修改的闭环。

| 步骤 | 节点名称 | 登录账号 | 关键填写表单数据与选择 | 期待流转结果 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | 正常发起与受理 | `admin` 发起<br>`case_acceptor1` 登记<br>`dept_leader2` 审批受理 | 正常填报，`dept_leader2` 选择 **受理 (true)** 并指定负责人为 `project_leader2`、辅助人为 `project_assistant2`。 | 案件正常流转至负责人与辅助人。 |
| **2** | 缴费确认 | `project_leader2`| 完成缴费登记节点，点击确认缴费。 | 案件主流程进入“编制内部质量控制文件”。 |
| **3** | 起草质控文件 | `project_assistant2`| 辅助人起草 F类质量控制表单，点击【提交流转】。 | 流转给项目负责人审核。 |
| **4** | 负责人审核 | `project_leader2`| **审核通过且需要用章**: 选择 **`是 (true)`**。<br>点击【提交流转】。 | 进入“用章申请”子流程，首先流转给档案员初审。 |
| **5** | 档案员初审 | `archivist` | 初审用章申请，选择通过。 | 流转给盖章经办人。 |
| **6** | 盖章驳回 | `archivist`| **物理盖章完成**: 选择 **`否 (false)` (即退回)**。<br>填写退回理由：`发现报告存在错别字`。<br>点击【退回流转】。 | **触发回退**：用章子流程终止并抛出异常信号，主流程自动回退到“项目辅助人编制质控文件”节点。 |
| **7** | 重新起草与通过 | `project_assistant2`| 修改表单后重新提交。后续 `project_leader2` 重新审核，`archivist` 重新初审，`archivist` 这次选择 **完成盖章 (true)**。 | 成功进入档案员回传与后续归档流程。 |

### 场景 3：大额现场勘验与征求意见反馈异议路径
*   **测试目的**：针对大额项目（>5万），测试“现场勘验”、“初稿征求意见”及“听证会处理”环节。

| 步骤 | 节点名称 | 登录账号 | 关键填写表单数据与选择 | 期待流转结果 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | 发起大额案件 | `admin` | **项目金额**: 填写 `80000` (8万，大于5万标准)。正常提交。 | 经过收案与受理。 |
| **2** | 负责人决策勘验 | `project_leader2`| 在“初勘或接收材料”节点，**是否需要现场勘验**: 选择 **`是 (true)`**。 | 案件进入现场勘验节点分支。 |
| **3** | 现场勘验 | `project_assistant2`<br>`project_leader2`| 辅助人记录勘验信息（天气、地点等），负责人审核确认。 | 推进至“起草鉴定意见初稿”。 |
| **4** | 触发征求意见 | `project_assistant2`| 编制初稿完成后提交。由于金额>5万，系统自动进入“征求意见与异议处理”流转。 | 发出征求意见稿。 |
| **5** | 反馈收到异议 | `project_leader2`| **是否收到异议**: 选择 **`是 (true)`**。 | **触发分支**：进入“组织听证会或补充审查”节点。 |
| **6** | 听证与出具正式稿 | `project_leader2`| 记录听证会结果，采纳部分异议后出具最终鉴定意见书。 | 流转至质量审查、复核环节，最终盖章归档结案。 |

### 场景 4：撤案退费、终止文书及邮寄入库归档
*   **测试目的**：测试案件中期被委托方强行撤案、财务退费以及纸质档案邮寄寄出的全套清算逻辑。

| 步骤 | 节点名称 | 登录账号 | 关键填写表单数据与选择 | 期待流转结果 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | 准备半途案件 | `admin` | 发起一个新案（例如标题 `测试撤案案件`），流转两步后搁置不管（使其处于“办理中”）。 | 获得一个活跃案件。 |
| **2** | 独立发起撤案 | `case_acceptor1` | 在侧边栏找“收到撤案函”新建。<br>**关联案件**: 选择刚才的 `测试撤案案件`。<br>填报理由提交。 | 流转至该案的负责人处理撤案。 |
| **3** | 判断退费 | `project_leader2`| **是否需要退费**: 选择 **`是 (true)`**。<br>点击【退费提交流转】。 | **触发跳转**：自动开启 `refund` (退费) 子流程。 |
| **4** | 财务退款执行 | `project_leader2`<br>`archivist`<br>`finance` | 负责人确认合同变更 -> 档案员登记退费材料 -> 财务执行打款 (**退款已操作完成: `true`**)。 | **触发跳转**：自动进入 `terminate-appraisal` (终止鉴定) 子流程。 |
| **5** | 终止文书与用章 | `project_assistant2`<br>`project_leader2` | 辅助人起草终止文书 -> 负责人审核通过并申请用章 -> 走用章流程。 | 用章完毕后，自动进入 `archive` (归档) 子流程。 |
| **6** | 邮寄移交与入库 | `archivist`<br>`case_acceptor1`<br>`center_archivist`| 档案员选择 **`入库方式：邮寄入库`** -> 邮寄员填快递单号 -> 中心档案员确认入库。 | 撤案子流程全部结束。 |

### 场景 6：案件暂停与恢复/终止
*   **测试目的**：测试案件推进中因外部原因（如等证据）暂停，后期恢复或终止的控制。

| 步骤 | 节点名称 | 登录账号 | 关键填写表单数据与选择 | 期待流转结果 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | 准备半途案件 | 任意账号 | 正常推进一个主案件至任意中间节点。 | 案件正常办理中。 |
| **2** | 申请暂停 | `project_leader2`| 在该案件的详情页操作栏，点击 **发起暂停申请**。填报原因并提交。 | 暂停子流程流转给部门负责人审批。 |
| **3** | 审批驳回/修改 | `dept_leader2`<br>`project_leader2` | `dept_leader2` 选择驳回 -> `project_leader2` 补充材料重新提交。 | 任务流转回部门负责人。 |
| **4** | 审批解冻/终止 | `dept_leader2` | 若选择 **`恢复办理`**：案件解冻继续原进度。<br>若选择 **`终止鉴定`**：直接触发并进入 `terminate-appraisal`（终止文书）流程，案件彻底终结。 | 测试两条分支均可。 |
