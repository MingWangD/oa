## 1. 核对结论

当前工作方向正确：先建设流程、表单、文件、知识库、审计和归档等公共平台能力，再逐条落实司法鉴定高保真流程。

但第四阶段不能停留在“20 个流程 / 19 个表单已导入平台配置”。后续验收必须以《完整 OA 系统重构需求规格说明书.docx》《司法鉴定系统使用手册.docx》和细化流程图为准，逐条确认：

- 节点角色是否独立。
- 输入文件、输出文件、注意事项是否进入表单/流程配置。
- 条件分支是否真实判断。
- 退回目标是否符合流程图。
- 并行任务、子流程、流程关联是否可运行。
- 表单、附件、意见、文书版本是否按案件号自动归档。
- 审计日志是否覆盖发起、保存、转交、退回、审批、预览、下载、归档和权限变更。

## 5. 首条流程：收到委托书高保真校准清单

### 5.1 手册确认的节点顺序

1. 任何授权用户新建“收到委托书”流程。
2. 发起者填写表单信息和附件，附件不能重复。
3. 转交收案员登记。
4. 收案员登记并检查后，转交部门负责人审阅。
5. 部门负责人确认委托审查是否受理，并指定项目负责人。
6. 项目负责人确定是否进行初步勘验，并指定项目辅助人。
7. 下一步骤允许并行或条件流转：
   - 告知项目辅助人：项目辅助人收到通知。
   - 收案员材料接收：可选择。
   - 初步勘验：当确认进行初步勘验时选择。
   - 发交费通知书及相关函件：当不进行初步勘验时选择。
   - 不予受理：当委托审查不受理时选择。

### 5.2 必须补齐的表单字段

| 字段组 | 字段 |
|---|---|
| 流程基础 | 流水号、流程名、发起人、发起日期、项目编号 |
| 委托信息 | 快递单号、收件日期、立案日期、委托人、案件号、承办法人、确定机构方式、确定机构时间 |
| 案件信息 | 鉴定类别、原告/申请人、被告/被申请人、项目紧急程度、线上/线下、项目金额、鉴定事项 |
| 附件 | 项目信息汇总表、委托附件；禁止同一表单重复附件 |
| 受理决策 | 是否进行初步勘验、委托审查是否受理、部门负责人、指定项目负责人、指定项目辅助人 |

### 5.3 必须补齐的运行规则

- 流程名称应由案件号、委托人等表单字段动态生成。
- 转交前必须校验必填字段、附件和下一步条件。
- 主办人和经办人必须分离；经办人默认只读。
- 退回必须填写原因，且只能退回到流程配置允许的目标。
- 输出文件在节点完成时固化版本并自动进入案件知识库。
- 每个节点完成时，表单、附件、意见和流程日志按案件号自动归档。
- 收到异议函、收到出庭通知允许主流程结束后独立发起，但必须关联原案件和历史流程。
