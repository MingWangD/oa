# 司法鉴定系统使用手册验收矩阵

更新时间：2026-06-14

## 1. 验收依据

本项目当前验收依据为：**《司法鉴定系统使用手册.docx》及 `docs/judicial-appraisal-flowchart-baseline.md` 登记的主流程图、细化流程图。**

验收流程范围按 `docs/judicial-appraisal-manual-based-requirements-v2.md` 第 9 章 9.1 至 9.19 执行，共 **19 个验收流程**。后续逐流程跑通应主要对照 `docs/flowcharts/` 中的流程图；流程图中存在但使用手册未列明的流程（如案件暂停）先作为差异项记录，不直接扩展当前必要验收指标。

2026-06-14 反查校准：本矩阵中的 `[x]` 必须代表手册级真实页面验收完成。后端运行时测试、Service 直调、MockMvc/API、流程配置存在、入口可见、子流程被触发，均只能作为部分证据。若缺非 admin 业务账号浏览器/Playwright 验证、附件、办理意见、流程日志、待办办理、办结只读、工作查询详情和必要分支，则状态必须为 `[~]` 或 `[ ]`。同日主干实例 `2758` 已打穿收案至归档/工作查询闭环；由于节点办理为真实业务账号 API 辅助提交，仍不能升为 `[x]`。

## 5. 下一步重点

1. **先复验 10 个业务账号登录与菜单权限**：不能跳过非 admin 账号基础能力。
2. **补 9.16 财务报销**：`finance` 账号入口已验证，仍需独立流程运行级测试和真实页面验收。
3. **补 9.15、9.17、9.18、9.19 页面闭环**：后端测试只能算部分证据，需按待办、办结、查询、附件、意见、日志、归档逐项验收。
4. **补公共功能缺口**：数据报表、批量下载、流程图视图、会签意见、经办人只读通知模型。
5. **知识库验收**：验证归档文件在知识库页面的展示、搜索和下载。
6. **保持范围收敛**：不把合同、CRM、项目、人资、行政等非手册范围扩展为必要目标。
