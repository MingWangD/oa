# 协作指引与愿景对齐

## 🌟 项目愿景
本项目旨在构建一个高度专业、规范、自动化的**司法鉴定业务管理系统**。通过现代化技术栈重构，完美还原并增强《司法鉴定系统使用手册》中的业务逻辑，为司法鉴定所提供生产级的数字化支撑。

## 🧭 核心协作原则
- **业务第一**：任何功能的实现或修改必须以《司法鉴定系统使用手册》为最高依据。
- **架构一致性**：坚持使用动态表单与流程引擎驱动，避免硬编码业务逻辑。
- **术语规范**：统一使用手册标准术语（如：收案员、项目负责人、财务人员），禁止引入非业务相关的 IT 术语到 UI 层。
- **闭环思维**：关注从发起、办理到归档、报表的完整业务闭环。

## 📚 知识库与参考
1. **最高依据**：`docs/司法鉴定系统使用手册.docx`
2. **业务底座**：`judicial-appraisal-backend` (Spring Boot) & `judicial-appraisal-frontend` (Vue 3)
3. **本地进度**：查阅 `GEMINI.md` 获取最新的业务实现进展。

## 🚀 业务领域映射
- **收案阶段**：涵盖委托接收、审查、初步勘验、缴费通知。
- **实施阶段**：涵盖质控编制、现场勘验、材料接收、送审稿编制。
- **结案阶段**：涵盖意见书出具、法函处理、撤案/退费处理、最终归档。
- **支撑管理**：涵盖报销流程、用章申请、统计报表、知识库检索。

## ⚠️ 严禁行为
- 禁止在 UI 或代码中泄露敏感测试数据。
- 禁止引入与司法鉴定无关的通用 OA 冗余模块。
- 除非用户明确要求，否则不要在 README 或公开文档中保留底层测试/验证细节。
