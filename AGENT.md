# Codex 协作约定 (2026-06-12 更新)

## 用户使用习惯

- 每次开始任务前，必须先阅读项目根目录下的 `GEMINI.md`，以获取最新的开发进度及当前上下文对齐信息。
- 每次实现、测试或验收司法鉴定功能前，必须对照 `docs/judicial-appraisal-master-checklist.md`。
- 当前唯一必要目标是：**严格按照《司法鉴定系统使用手册.docx》完美实现司法鉴定管理系统全部功能。**
- 不再以“完整 OA”为当前必要目标。合同、CRM、项目、人资、行政等模块已标记为历史非必要范围。
- 每次修改代码后，运行可用的前后端测试和构建。

## 需求优先级

需求来源发生冲突时，严格按照以下顺序处理：

1. **`docs/司法鉴定系统使用手册.docx`**
2. **`docs/judicial-appraisal-flowchart-baseline.md`** 及其登记的 `docs/flowcharts/` 主流程图、细化流程图
3. **`docs/judicial-appraisal-manual-based-requirements-v2.md`** (或相关导出的文本)
4. 当前项目中的现有代码与配置
5. `docs/` 目录下的旧 Markdown 文档 (仅作历史参考)

## 最高执行指示

- **产品范围**：当前产品范围严格限定为《司法鉴定系统使用手册》所列内容。
- **验收标准**：以手册列明的 3.1-3.18 流程章节 + 独立财务报销流程（共 **19 个验收流程**）为准。
- **流程图基准**：后续运行级验收主要按 `docs/judicial-appraisal-flowchart-baseline.md` 登记的主流程图与细化流程图跑通；流程图不能覆盖或替代手册范围，但必须用于节点、分支、退回和产物路径校验。
- **非必要范围**：合同、CRM、项目、人资、行政、公文、仓库等通用 OA 模块不属于当前必要验收范围。
- **目标对齐**：当前验收标准是“完美符合使用手册”，而不是“超出使用手册”或“通用 OA 重构”。

## 司法鉴定必须实现的能力 (对齐手册)

- **19 个验收流程**：从 3.1 收到委托书到 3.18 用章流程，加上财务报销。
- **通用办理页**：支持会签意见、主办/经办权限分离、转交、退回、流程图、附件管理。
- **自动化能力**：必填校验、附件查重、产物自动归档知识库、审计全覆盖。
- **专属角色**：部门负责人、项目负责人、项目辅助人、档案管理员、中心档案管理员、技术负责人、审阅所长、综合业务部、财务、收案员等。
- **查询与统计**：符合手册 2.3 节和 2.4 节要求的查询与报表展示。
