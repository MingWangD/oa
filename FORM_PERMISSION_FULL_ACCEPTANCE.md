# 表单权限与动态标红全量修改验收测试说明

更新时间：2026-06-25

## 一、验收基准

本轮表单权限与动态标红以《使用手册》第三章“流程流转详细介绍”的截图为准。页面上线形态必须符合以下原则：

- 截图中带红星的字段，当前系统也必须显示红星，并且输入框、下拉框、日期框等控件需要同步标红。
- 截图中没有红星的字段，不应因为兜底规则被误标为必填。例如“收到委托书”中的“快递单号”按截图为可填写但非必填。
- 当前节点负责填写的字段才可编辑；其他流程角色负责的字段应只读、不可误改、不可显示必填红星。
- 只读字段、隐藏字段不参与当前节点必填校验。
- 前端显示与后端提交校验统一使用同一条公式：

```text
显示必填红星 = required === true && readonly !== true && hidden !== true
```

## 二、已修复的明确偏差

### 2.1 收到委托书“快递单号”误标必填

问题：

- 前一版兜底规则把 `expressNo`（快递单号）加入了 `INIT_FILL` 和 `CLERK_REGISTER` 的必填集合。
- 这会导致新建“收到委托书”时，快递单号被错误显示红星和红框。
- 对照使用手册 3.1 截图，快递单号没有红星，因此不应必填。

修复：

- 后端运行时兜底集合移除 `expressNo`。
- 前端草稿态兜底集合移除 `expressNo`。
- 表单源配置将 `expressNo.required` 调整为 `false`。
- 数据库修复脚本将既有节点配置里的 `expressNo` 统一修复为：

```json
{
  "readonly": false,
  "required": false
}
```

涉及文件：

- `judicial-appraisal-backend/src/main/java/com/example/judicialappraisal/workflow/service/FieldAccessRules.java`
- `judicial-appraisal-frontend/src/views/CaseDetailView.vue`
- `judicial-appraisal-backend/src/main/java/com/example/judicialappraisal/platform/service/JudicialConfigImportService.java`
- `scripts/fix_form_permissions.js`
- `judicial-appraisal-backend/src/test/java/com/example/judicialappraisal/workflow/service/FieldAccessRulesTests.java`
- `FORM_PERMISSION_REQUIRED_RULES.md`

本地数据库已执行：

```bash
node scripts/fix_form_permissions.js --apply
node scripts/fix_form_permissions.js --verify
```

验收结果：

- `INIT_FILL`：`expressNo` 为可填写、非必填。
- `CLERK_REGISTER`：`expressNo` 为可填写、非必填。
- 最新发布版 `received-entrust` 表单字段源配置中，`expressNo.required=false`。

## 三、第三章流程截图基准清单

### 3.1 收到委托书

流程概要：

- 任意用户新建流程，选择“收到委托书”。
- 发起者填写委托信息。
- 转收案员登记。
- 收案员登记并检查后转部门负责人审阅。
- 部门负责人确认委托审查是否受理，并指定项目负责人。
- 项目负责人确认是否初步勘验，并指定项目辅助人。
- 后续按条件并行或分支进入：告知项目辅助人、收案员材料接收、初步勘验、发缴费通知书及相关函件、不予受理。

验收重点：

- “快递单号”可填写但不显示红星、不标红。
- 收件日期、立案日期、委托人、案件号、承办法人、确定机构方式、确定机构时间、鉴定类别、原告/申请人、被告/被申请人、鉴定事项等截图带星字段需要显示红星与红框。
- 部门负责人、项目负责人、项目辅助人等后续审批/指派字段不得在发起者或收案员阶段误标红。

### 3.2 初步勘验

流程概要：

- 项目负责人确认初步勘验时间、人员，电话询问委托人是否参加初步勘验。
- 项目负责人确认以上工作后，上传现场工作方案，转交项目辅助人作勘验准备。
- 项目辅助人上传仪器设备库出入表、仪器设备使用记录，确认初步勘验准备完成。
- 项目负责人审核材料，不符合要求可退回；审核通过后确认是否具备鉴定条件。
- 具备鉴定条件进入“发交费通知书及相关函件”；不具备则进入“终止鉴定”。

验收重点：

- 项目负责人节点只开放项目负责人决策和审核字段。
- 项目辅助人节点只开放设备、方案、准备完成等执行字段。
- 审核结果类字段仅在对应审核角色节点标红。

### 3.3 发交费通知书及相关函件

流程概要：

- 流程到达项目负责人，基础信息从父流程同步，无需重复填写。
- 项目负责人转项目辅助人编制相关函件。
- 项目辅助人上传相关文件，完成后交项目负责人审阅。
- 项目负责人审核通过后交档案管理员申请盖章。
- 档案管理员确认盖章后提交用章申请子流程。
- 档案管理员上传需盖章文件；用章申请子流程完成后交项目辅助人线下盖章。
- 项目辅助人上传盖章后的电子版，转项目负责人。
- 项目负责人确认是否缴费：若是，进入“编制内部质量控制文件”；若否，进入“终止鉴定”。

验收重点：

- 函件上传、盖章申请、盖章扫描件、是否缴费等字段只在对应节点标星。
- 父流程同步的基础字段应只读。

### 3.4 编制内部质量控制文件

流程概要：

- 由“发缴费通知书”流转至此，最初节点到达项目辅助人。
- 项目辅助人编制并上传质量控制文件。
- 系统按合同金额和中心格式判断是否为 F 类项目。
- 无论是否 F 类，流程先到项目负责人审核。
- F 类项目需部门负责人审核。
- 档案管理员确认盖章并提交用章申请。
- 项目辅助人线下盖章后上传扫描版，转项目负责人。
- 项目负责人确认是否需要现场勘验，并按条件进入后续流程。

验收重点：

- 项目辅助人只能填报质量控制文件相关字段。
- F 类判定、项目负责人审核、部门负责人审核、档案管理员盖章字段应按节点分别标星。
- 后续分支选择字段只在项目负责人决策节点开放。

### 3.5 现场勘验

流程概要：

- 由“编制内部质量控制文件”流转至项目辅助人。
- 项目辅助人编制和上传现场工作方案，提交项目负责人审核。
- 项目金额小于 15 万时，由项目负责人审核；大于 15 万时，依次由项目负责人、技术负责人、部门负责人审核。
- 项目负责人再转项目辅助人填写仪器设备相关表。
- 项目辅助人上传仪器设备出入库记录表、仪器设备使用记录，确认现场勘验工作准备完成。
- 项目负责人对上传材料审核，并根据是否补充材料、是否满足报告编制条件、是否需要征求意见稿、是否退费等条件流转。

验收重点：

- 现场工作方案、设备记录、审核结果、后续分支字段按节点角色分别可写。
- 多级审核字段不得提前在项目辅助人节点显示必填。

### 3.6 材料接收与返还

流程概要：

- 由“编制内部质量控制文件”或“现场勘验”流转至此，起始节点一般为项目负责人。
- 项目负责人选择材料接收与返还类型。
- 若委托方直接提供，选择同事为主办人，由该同事上传材料。
- 若需要补充材料，编制并上传补充鉴定材料通知书，进入“发缴费通知书及相关函件”流程。
- 项目负责人确认材料后，转项目辅助人作材料接收与登记。
- 项目辅助人填写材料介质类别、材料存放地址，并确认材料是否需要归还。
- 需要归还则填写材料接收及返还登记表，同步流转材料返还；无需归还则交档案管理员保管并进入归档。
- 同时项目负责人确认是否满足报告编制条件，决定进入征求意见稿、送审稿、退费或终止鉴定。

验收重点：

- 类型选择、材料上传、材料介质类别、存放地址、是否归还、报告条件判断字段按节点标星。
- 材料相关字段和项目负责人分支决策字段要互相隔离权限。

### 3.7 鉴定意见书征求意见稿送审编制

流程概要：

- 由其他流程关联发起，初始节点一般为项目负责人。
- 项目负责人转交项目辅助人编制初稿。
- 项目辅助人编制并上传初稿，转项目负责人审核。
- 项目负责人审核通过后转技术负责人审核，不符合可退回项目辅助人。
- 技术负责人审核通过后转部门负责人审核，不符合可退回。
- 部门负责人审核通过后转项目负责人。
- 项目负责人上传最终版送审稿，进入“出具征求意见稿”。

验收重点：

- 初稿上传字段只在项目辅助人节点必填。
- 项目负责人、技术负责人、部门负责人审核结果分别只在对应节点必填。
- 最终版本上传只在项目负责人最终确认节点必填。

### 3.8 鉴定意见书送审稿编制

流程概要：

- 一般由其他流程关联发起，初始节点一般为项目负责人。
- 项目负责人确认项目辅助人填写内容后交项目辅助人。
- 项目辅助人编写并上传鉴定意见书初稿。
- 项目负责人、技术负责人、部门负责人依次审阅。
- 审核通过后，项目负责人上传最终版本送审稿，进入“出具鉴定意见书”。

验收重点：

- 项目辅助人的草稿上传字段、各级审核结果、最终版本上传字段按节点分开标星和授权。

### 3.9 出具鉴定意见书

流程概要：

- 一般由其他流程关联发起，初始节点一般为项目负责人。
- 项目负责人可修改鉴定意见书，转交项目辅助人。
- 项目辅助人上传鉴定人承诺书、司法鉴定复核意见。
- 项目负责人审核通过后，转档案管理员申请盖章和开票。
- 档案管理员确认盖章申请，上传中电投系统编号登记表纸质扫描件。
- 可同步进入用章申请表和申请开票流程。
- 开票申请完成后节点结束。
- 用章申请表节点填写并上传文件后，项目辅助人线下盖章并上传盖章扫描件，然后归档。

验收重点：

- 承诺书、复核意见、审核结果、盖章申请、开票资料等字段必须按节点角色标星。

### 3.10 出具征求意见稿

流程概要：

- 项目辅助人编制并上传《鉴定说明函》。
- 项目负责人审核材料，符合要求则审核结果填写“通过”，转档案管理员。
- 档案管理员同意后，在申请盖章处填写“同意”，上传《鉴定意见书（征求意见稿）》并提交用章申请。
- 档案管理员填写相关信息，上传鉴定说明函和征求意见稿，提交下一步。
- 项目辅助人线下盖章后上传扫描件，选择后续流程：归档和材料寄出。
- 项目辅助人材料寄出后填写材料邮寄快递单号，转项目负责人等待反馈。
- 项目负责人根据实际结果填写是否收到异议函。
- 若收到异议函，进入“收到法院其他函件（含异议函）”；若未收到，进入“出具鉴定意见书”。

验收重点：

- 《鉴定说明函》上传、审核结果、申请盖章结果、盖章扫描版、材料邮寄快递单号、是否收到异议函按节点分别标星。

### 3.11 收到法院其他函件（含异议函）

流程概要：

- 可能由“出具鉴定意见书征求意见稿”自然流转，也可在出具意见书结束后手动新建。
- 手动新建时，需要在流程关联区关联先前流程。
- 首先上传函件，然后转交项目负责人。
- 项目负责人判断是否为异议函。
- 若是，流转到“编制异议回复函”；若否，判断是否需要回复其他函件。
- 需要回复则由项目辅助人编制函件；无需回复则进入归档。
- 同意盖章时，进入用章申请表子流程；盖章后回到项目辅助人。

验收重点：

- 收到函件、是否异议函、异议回复函、是否需要回复其他函件、其他回复函、申请盖章结果等字段按节点标星。
- 非鉴定意见书相关分支进入时，应取消错误勾选的“出具鉴定意见书”子流程。

### 3.12 收到出庭通知

流程概要：

- 可单独发起，也可关联原流程。
- 项目负责人确认函件内容和项目编号。
- 项目负责人确认内容无误后，流转到发送出庭费缴费通知书及调档流程。
- 档案管理员上传档案借阅登记表，判断是否存放至中心档案室。
- 若中心档案室不在本系统，需要线下完成，系统仅作留档。
- 完成档案取阅后流转至项目负责人。
- 项目负责人完成出庭准备文件后，上传相关文件并进入出庭状态。
- 出庭后流程继续流转给项目负责人，对出庭材料进行整理。
- 整理后所有信息归档。

验收重点：

- 出庭申请书、项目编号确认、档案借阅登记表、是否存放中心档案室、出庭准备文件、出庭材料整理等字段按节点标星。

### 3.13 不予受理

流程概要：

- 出现在收到申请书后判断不满足受理条件。
- 流程先到项目辅助人，由项目辅助人编制不予受理通知书。
- 项目负责人审核。
- 审核通过后转档案管理员盖章；审核未通过可退回。
- 档案管理员盖章后转项目辅助人。
- 项目辅助人上传盖章扫描版后归档。

验收重点：

- 不予受理通知书电子版、盖章纸质版、审核是否通过、盖章是否通过等字段按节点标星。

### 3.14 收到撤案函

流程概要：

- 司法鉴定流程中随时可能收到撤案函，此时司法鉴定流程将终止。
- 收到撤案函后，转项目负责人。
- 项目负责人判断是否退费。
- 不需退费则转终止鉴定；需要退费则进入退费流程。

验收重点：

- 收到撤案函人员、收到时间、是否退费等字段按节点标星。
- 撤案流转后的终止或退费分支选择不得提前开放给其他角色。

### 3.15 退费流程

流程概要：

- 由项目负责人再次选择，若不需退费则进入终止鉴定；若需退费则流转至档案管理员。
- 档案管理员上传对应文件，进入申请合同变更。
- 档案管理员确认合同变更通过后，记录是否已经确认收入。
- 申请退费成果后，进入打款流程。
- 打款成功后，进入终止鉴定。

验收重点：

- 是否退费、退费申请文件、合同变更系统文件、是否已经确认收入、申请合同变更结果、申请退费结果、打款结果等字段按节点标星。

### 3.16 终止鉴定

流程概要：

- 当不具备鉴定条件、收到撤案函、未缴费等情况发生时进入。
- 初始节点一般为项目负责人。
- 项目负责人转项目辅助人编写鉴定终止函，也可由项目负责人上传，但非必填。
- 项目辅助人填写并上传鉴定终止函或确认函，交项目负责人审核。
- 项目负责人审核材料，通过后转档案管理员盖章。
- 档案管理员确认盖章后，申请盖章结果填写“通过”，提交用章申请。
- 填写和上传材料后，转项目辅助人线下盖章。
- 项目辅助人上传盖章扫描件后，进入归档。

验收重点：

- 终止函上传、审核结果、申请盖章结果、盖章扫描件字段按节点标星。

### 3.17 归档

流程概要：

- 一般由其他流程流转至此，到达档案管理员。
- 档案管理员上传项目档案。
- 扫描纸质文档，填写电子归档地址，并确认是否需要邮寄。
- 若需要邮寄，填写邮寄快递单号后提交中心档案管理员审核。
- 若无需邮寄，直接提交中心档案管理员审核。
- 中心档案管理员审核通过后，转相关中心档案管理员，存放中电投档案室。
- 流程结束。

验收重点：

- 项目档案、电子归档地址、是否需要邮寄、邮寄快递单号、中心档案审核结果等字段按节点标星。

### 3.18 用章申请表
