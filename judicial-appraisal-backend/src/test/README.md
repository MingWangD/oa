# 后端测试注意事项

## 测试账号角色复原

部分工作流联调或手工验收为了快速走通全流程，可能会临时给测试账号增加多个角色。测试结束后必须立即恢复账号的标准角色，避免把临时扩权状态留在本地数据库或导出的 `judicial_appraisal_full_dump.sql` 中。

重点检查以下账号：

- `admin`：只保留 `系统管理员` 角色。
- `tech_leader`：只保留 `技术负责人` 角色。

如果测试中临时给上述账号增加了收案员、项目负责人、项目辅助人、部门负责人、档案管理员、财务、申请人、盖章经办人、邮寄人员等角色，测试完成后请立刻移除这些临时角色，再进行截图、导出数据库或提交测试数据。

推荐流程：

1. 测试前记录临时扩权的账号和角色。
2. 测试完成后进入用户管理页面恢复角色。
3. 刷新页面确认角色列表已经回到基线。
4. 再导出数据库、更新测试说明或提交代码。
