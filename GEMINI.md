# 项目核心能力与进展摘要

本文档由 Gemini CLI 维护，作为司法鉴定管理系统重构项目的核心备忘录，重点记录系统架构、业务逻辑实现以及核心进展。

## 🏗️ 系统核心架构
- **后端底座**：基于 Spring Boot + MyBatis-Plus，采用领域驱动设计的精简模式。
- **动态平台**：
    - **流程引擎**：自主研发的轻量级运行时引擎，支持流程版本化、子流程挂起与唤醒、并行网关。
    - **表单设计器**：支持动态字段定义、必填校验规则及前端高保真渲染。
- **鉴权体系**：标准 RBAC 权限模型，已实现行级数据权限隔离与多角色切换。
- **文件与知识库**：集成自动归档能力，流程产物在节点完成时自动存入知识库，支持批量下载与审计。

## 💼 业务实现进展 (2026-06-14)
- **司法鉴定核心流程**：
    - 已完成全部 19 个业务流程的后端逻辑与表单配置（涵盖 3.1 收到委托书至 3.18 用章流程，含财务报销）。
    - 实现了高保真业务流转：支持退回补充、多级审核、跨部门协同。
- **公共功能模块**：
    - **报表中心**：已实现实时数据统计与展示，支持多维度过滤、分页设置及导出样板。
    - **工作台**：统一的待办/办结入口，支持基于角色的菜单与功能过滤。
    - **知识库**：完整的归档文档树形管理，支持在线预览与版本追溯。

## 🛠️ 关键技术决策
1. **去“通用化”倾向**：项目已明确收敛至司法鉴定业务核心，移除了非必要的 CRM、合同等冗余设计，确保系统运行效率。
2. **术语对齐手册**：全系统（含代码类名、数据库角色名、UI 字样）已完成与《司法鉴定系统使用手册》的 1:1 对齐，确保业务人员无缝切换。
3. **前端现代化**：采用 Vue 3 + TypeScript，移除了所有开发期的测试用语，提供生产级的交互体验。

## 📈 后续演进方向
- **深度业务分析**：增加基于 AI 的案件数据分析与质量控制预警。
- **接口集成优化**：进一步增强与外部法律/法院系统的对接能力。
- **生产环境收口**：完善生产部署文档与性能调优。
