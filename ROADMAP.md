# 司法鉴定管理系统后续开发路向 (Roadmap)

根据《司法鉴定系统使用手册》及当前项目 `checklist.md` 的对齐情况，制定如下后续任务安排。

## 🎯 核心目标
将所有业务流程从后端逻辑就绪（[~]）提升至**生产级全链路验收就绪**（[x]），确保每一个操作节点均符合司法鉴定实务标准。

## 🛠️ 后续任务规划

### 第一阶段：公共能力补齐与加固 (2026-06-15 ~ 2026-06-20)
- **[ ] 权限矩阵全量验收**：
    - 验证 10+ 个业务角色（收案员、鉴定人、复核人、签发人等）在真实浏览器中的菜单可见性与数据隔离。
    - 修复非 admin 账号在进入部分复杂流程节点时的权限报错。
- **[ ] 知识库功能闭环**：
    - 开发**手动上传**功能，支持非流程产物的归档。
    - 优化文件夹点击语义，支持中心/边缘存储的区分显示。
- **[ ] 流程增强**：
    - 在办理页面增加**流程图查看**（只读模式）。
    - 完善“**保存暂存**”逻辑，允许用户在不提交任务的情况下保存当前表单进度。

### 第二阶段：核心业务流转验收 (2026-06-21 ~ 2026-07-05)
- **[ ] 全流程节点证据补齐**：
    - 对 9.1 至 9.19 共 19 个业务流程进行全量跑通。
    - 每一节点需在 `checklist.md` 中记录真实页面的输入、附件提交、意见签署及日志生成证据。
- **[ ] 特殊分支开发**：
    - 实现 9.9 流程中的**开票分支**逻辑。
    - 实现 9.15/9.17 的**退费与终止**流程与父流程的状态同步。

### 第三阶段：档案与审计专项 (2026-07-06 ~ 2026-07-15)
- **[ ] 档案库管理页面**：
    - 新建独立的“档案库”视图，支持按物理库房、柜号进行数字化管理。
    - 对接 9.18 归档流程的最终入库确认环节。
- **[ ] 审计日志可视化**：
    - 为管理员提供文书预览、下载的专项审计记录查询页面。

## 📈 验收标准
1. **真实页面覆盖**：所有 `checklist.md` 中的 `[~]` 必须转为 `[x]`。
2. **零错误闭环**：从“收到委托书”到“归档/用章”的全链路不出现 500 错误或数据不同步现象。
3. **手册 1:1 对齐**：操作术语、文书模板样式完全符合司法鉴定业务规范。
